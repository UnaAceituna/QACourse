Тест-дизайн - один из первоначальных этапов тестирования по, этап планирования и проектирования тестов. Он представляет собой продумывание и написание test case в соответствии с требованиями проекта, критериями качества будущего по и финальными целями тестирования.
цели:
-обеспечить покрытие тестами функционала приложения. Тестов должно быть минимально достаточно
задачи
-проанализировать требования к продукту
-оценить возможные риски при исп продукта
-написать достат миним колво тестов

1. техника - разделение на классы эквивалентности.
Позволяет минимизировать число тестов, не создавая сценарий для каждого возможного значения, но выбрав только одно значение из целого класса и приняв за аксиому, что для всех значений в данной группе результат будет аналогичным
2. Техника анализ граничных значений - основана на предположении, что большинство ошибок могут возникнуть на границах эквив классов
3. Попарное тестировани. Суть метода в том, что каждое значение каждого проверяемого параметра должно быть протестировано на взаимодействие с каждым значением всех остальных параметров. Максимальное покрытие при минимальном необходимом наборе сценариев.
4. Тхника перехода состояний. Тест черного ящика, в котором тестировщик видит поведение тестируемого по для различных входных условий в последовательности
5. use cese техника. Сценарии, описывающие то как юзер пользуется системой для достижения целей. Варианты использования с точки зрения пользователя, а не системы.
6. Тестирование на основе рисков или предугадывание ошибок. Основную роль игрет опыт и интуиция qa
